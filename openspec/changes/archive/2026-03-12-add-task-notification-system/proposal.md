## Why

用户完成任务后缺乏即时反馈和提醒机制。当前系统仅支持静态任务展示，无法实时通知用户任务状态变更，也无法提醒即将到期的任务。这导致用户可能错过重要任务的截止日期，影响工作效率和任务管理体验。

## What Changes

### 1. 任务卡片勾选功能完善
- 在 TaskCard 组件中实现完整的任务完成/取消完成交互
- 点击复选框即可快速切换任务状态，无需打开编辑对话框
- 添加完成状态变更的视觉反馈（动画、样式变化）

### 2. 消息数量实时同步
- 任务状态变更后，侧边栏导航栏的各分类任务数量自动更新
- 包括：全部任务、今日任务、过期任务、即将到期、无期限、已完成
- 确保用户随时看到准确的统计数据

### 3. 实时通知系统（WebSocket）
- 实现基于 WebSocket 的实时通信基础设施
- 即将到期任务每隔1小时自动提醒（截止日期前24小时内的任务）
- 任务完成/取消完成时发送通知
- 前端通知中心展示和管理所有通知

## Capabilities

### New Capabilities

- `websocket-realtime`: WebSocket 实时通信基础设施，支持服务端主动推送消息到客户端，包括连接管理、心跳检测、重连机制

- `notification-system`: 通知系统，管理应用内通知的创建、展示、已读状态和历史记录，支持多种通知类型（任务提醒、状态变更等）

- `task-reminder`: 任务提醒服务，定时检查即将到期的任务并通过通知系统提醒用户

### Modified Capabilities

- `task-display`: 新增任务完成状态快速切换功能，用户可通过复选框直接标记任务完成状态

- `sidebar-navigation`: 新增任务统计数量的实时更新机制，任务状态变更后自动刷新各分类数量

## Impact

### 后端
- 新增 WebSocket 服务（使用 ws 或 socket.io）
- 新增通知相关的数据模型和 API
- 新增定时任务调度器（检查即将到期任务）
- 修改任务更新 API，在状态变更时触发通知

### 前端
- 新增 WebSocket 客户端连接管理
- 新增通知中心组件
- 修改 TaskCard 组件，完善勾选交互
- 修改 Sidebar 组件，实现数量实时更新
- 修改 Task Store，集成 WebSocket 消息处理

### 数据库
- 新增 `notifications` 表存储通知记录

### 依赖
- 后端新增：`ws` 或 `socket.io`、`node-cron`（定时任务）
- 前端新增：`socket.io-client`（如果使用 socket.io）
