# Task Reminder Specification

## Capability

任务提醒服务，定时检查即将到期的任务并通过通知系统提醒用户。

## ADDED Requirements

### REQ-001: 定时检查机制

**优先级**: P0

系统 MUST 定时检查即将到期的任务并发送提醒。

**验收标准**:
- [ ] 每小时执行一次检查任务
- [ ] 检查截止日期在未来24小时内的未完成任务
- [ ] 每个任务每天最多提醒一次（避免重复）
- [ ] 使用 node-cron 实现定时任务

#### Scenario: 定时检查触发
- **WHEN** 到达整点时间（如 10:00、11:00）
- **THEN** 执行任务到期检查
- **AND** 识别需要提醒的任务

#### Scenario: 检查即将到期任务
- **WHEN** 任务截止日期在24小时内
- **AND** 任务状态为未完成
- **THEN** 创建提醒通知
- **AND** 通过 WebSocket 推送给用户

### REQ-002: 提醒内容格式

**优先级**: P0

提醒通知 MUST 包含清晰的任务信息。

**验收标准**:
- [ ] 通知标题：任务到期提醒
- [ ] 通知内容：任务名称 + 剩余时间
- [ ] 包含任务 ID 以支持跳转
- [ ] 根据剩余时间调整紧迫程度

#### Scenario: 1小时内到期
- **WHEN** 任务将在1小时内到期
- **THEN** 通知内容显示"任务「XXX」将在XX分钟后到期"
- **AND** 通知标记为紧急

#### Scenario: 24小时内到期
- **WHEN** 任务将在24小时内到期（超过1小时）
- **THEN** 通知内容显示"任务「XXX」将在X小时后到期"

### REQ-003: 提醒去重机制

**优先级**: P1

同一任务 MUST NOT 在同一天重复提醒。

**验收标准**:
- [ ] 使用 Redis 记录已提醒的任务
- [ ] 缓存键：`reminder:{date}:{taskId}`
- [ ] 缓存过期时间：24小时
- [ ] 检查提醒前先验证是否已提醒

#### Scenario: 避免重复提醒
- **WHEN** 任务在今天已被提醒过
- **THEN** 本次检查跳过该任务
- **AND** 不创建重复通知

### REQ-004: 服务启动检查

**优先级**: P1

服务重启后 MUST 检查是否有需要立即提醒的任务。

**验收标准**:
- [ ] 服务启动时执行一次即时检查
- [ ] 检查是否有即将在1小时内到期的任务
- [ ] 立即发送提醒通知
- [ ] 避免服务重启导致遗漏提醒

#### Scenario: 服务重启后补救
- **WHEN** 服务启动
- **THEN** 立即检查1小时内到期的任务
- **AND** 发送提醒通知

## Technical Constraints

### TC-001: 定时任务
- 使用 node-cron，cron 表达式：`0 * * * *`（每小时整点）
- 定时任务在独立进程/模块中运行

### TC-002: 去重存储
- 使用 Redis 存储提醒记录
- Key 格式：`reminder:{YYYY-MM-DD}:{taskId}`
- TTL：24小时

### TC-003: 性能
- 检查任务查询优化，使用索引
- 大量任务时分批处理（每批100条）
