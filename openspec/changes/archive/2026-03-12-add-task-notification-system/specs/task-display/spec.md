# Task Display Specification

## Capability

优化任务展示方式，支持描述内容展示、卡片视图、看板视图和**任务完成状态快速切换**。

## ADDED Requirements

### REQ-004: 任务完成状态快速切换

**优先级**: P0

用户 MUST 能够通过任务卡片的复选框快速切换任务完成状态。

**验收标准**:
- [ ] 任务卡片显示复选框（左侧）
- [ ] 点击复选框立即切换完成状态
- [ ] 状态变更有视觉反馈（动画、样式变化）
- [ ] 已完成任务显示删除线样式
- [ ] 支持取消完成（再次点击复选框）

**UI规格**:
```
┌────────────────────────────────────────────────┐
│ ☑ 完成项目报告                          🔴 高优先级│  ← 已完成
│ ────────────────────────────────────────────   │
│ 需要包含Q4数据分析...                          │
└────────────────────────────────────────────────┘

┌────────────────────────────────────────────────┐
│ ☐ 准备周会材料                          🟡 中优先级│  ← 未完成
│ ────────────────────────────────────────────   │
│ 整理本周工作进度...                            │
└────────────────────────────────────────────────┘
```

#### Scenario: 标记任务完成
- **WHEN** 用户点击未完成任务卡片的复选框
- **THEN** 任务状态变更为已完成
- **AND** 卡片显示完成样式（删除线、淡化）
- **AND** 触发完成通知

#### Scenario: 取消任务完成
- **WHEN** 用户点击已完成任务卡片的复选框
- **THEN** 任务状态变更为未完成
- **AND** 卡片恢复正常样式
- **AND** 触发取消完成通知

### REQ-005: 任务状态变更乐观更新

**优先级**: P1

任务状态变更 MUST 立即反映在 UI 上，无需等待服务器响应。

**验收标准**:
- [ ] 点击复选框后 UI 立即更新
- [ ] API 请求在后台异步执行
- [ ] 请求失败时 UI 回滚并显示错误提示
- [ ] 请求成功后保持当前状态

#### Scenario: 乐观更新成功
- **WHEN** 用户点击复选框
- **THEN** UI 立即显示新状态
- **AND** 后台发送 API 请求
- **AND** 请求成功后保持状态

#### Scenario: 乐观更新失败回滚
- **WHEN** 用户点击复选框
- **THEN** UI 立即显示新状态
- **AND** API 请求失败
- **THEN** UI 回滚到原状态
- **AND** 显示错误提示"操作失败"
