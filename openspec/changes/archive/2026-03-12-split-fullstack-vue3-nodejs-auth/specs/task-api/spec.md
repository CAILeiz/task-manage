## ADDED Requirements

### Requirement: 用户可以创建任务
系统 SHALL 允许已登录用户创建任务，任务包含名称、描述、优先级、截止日期。

#### Scenario: 成功创建任务
- **WHEN** 已登录用户提供有效的任务信息调用创建任务 API
- **THEN** 系统创建任务并返回 201 状态码和任务详情
- **AND** 任务自动关联到当前登录用户

#### Scenario: 任务名称为空
- **WHEN** 用户提供空名称调用创建任务 API
- **THEN** 系统返回 400 状态码和错误信息 "任务名称不能为空"

#### Scenario: 未登录用户创建任务
- **WHEN** 未登录用户调用创建任务 API
- **THEN** 系统返回 401 状态码

### Requirement: 用户可以查询自己的任务列表
系统 SHALL 允许已登录用户查询属于自己的任务列表，支持分页和过滤。

#### Scenario: 获取任务列表成功
- **WHEN** 已登录用户调用获取任务列表 API
- **THEN** 系统返回该用户的所有任务列表
- **AND** 不包含其他用户的任务

#### Scenario: 按优先级过滤任务
- **WHEN** 已登录用户使用 priority=HIGH 参数调用获取任务列表 API
- **THEN** 系统仅返回高优先级的任务

#### Scenario: 按完成状态过滤任务
- **WHEN** 已登录用户使用 completed=false 参数调用获取任务列表 API
- **THEN** 系统仅返回未完成的任务

#### Scenario: 分页查询任务
- **WHEN** 已登录用户使用 page=1&limit=10 参数调用获取任务列表 API
- **THEN** 系统返回第 1 页的 10 条任务和总数量

### Requirement: 用户可以查询单个任务详情
系统 SHALL 允许已登录用户查询属于自己的单个任务详情。

#### Scenario: 获取任务详情成功
- **WHEN** 已登录用户使用有效的任务 ID 调用获取任务详情 API
- **THEN** 系统返回该任务的完整信息

#### Scenario: 访问不存在的任务
- **WHEN** 已登录用户使用不存在的任务 ID 调用获取任务详情 API
- **THEN** 系统返回 404 状态码

#### Scenario: 访问其他用户的任务
- **WHEN** 用户 A 尝试获取用户 B 的任务详情
- **THEN** 系统返回 403 状态码

### Requirement: 用户可以更新任务
系统 SHALL 允许已登录用户更新属于自己的任务。

#### Scenario: 成功更新任务
- **WHEN** 已登录用户提供有效的任务 ID 和更新信息调用更新任务 API
- **THEN** 系统更新任务并返回更新后的任务详情

#### Scenario: 更新其他用户的任务
- **WHEN** 用户 A 尝试更新用户 B 的任务
- **THEN** 系统返回 403 状态码

#### Scenario: 更新不存在的任务
- **WHEN** 已登录用户使用不存在的任务 ID 调用更新任务 API
- **THEN** 系统返回 404 状态码

### Requirement: 用户可以删除任务
系统 SHALL 允许已登录用户删除属于自己的任务。

#### Scenario: 成功删除任务
- **WHEN** 已登录用户使用有效的任务 ID 调用删除任务 API
- **THEN** 系统删除该任务并返回 200 状态码

#### Scenario: 删除其他用户的任务
- **WHEN** 用户 A 尝试删除用户 B 的任务
- **THEN** 系统返回 403 状态码

#### Scenario: 删除不存在的任务
- **WHEN** 已登录用户使用不存在的任务 ID 调用删除任务 API
- **THEN** 系统返回 404 状态码

### Requirement: 任务数据按用户隔离
系统 SHALL 确保每个用户只能访问自己的任务数据。

#### Scenario: 数据隔离验证
- **GIVEN** 用户 A 有 5 个任务，用户 B 有 3 个任务
- **WHEN** 用户 A 调用获取任务列表 API
- **THEN** 系统仅返回用户 A 的 5 个任务
- **AND** 不包含用户 B 的任何任务
