# Design: Frontend UI Optimization

## Context

### 当前状态

项目使用 Vue 3 + TypeScript + Element Plus + Pinia 技术栈，已有基础的 CSS Variables 主题系统，支持暗色模式。当前 UI 功能完整但视觉质感与 Linear、Notion 等顶尖产品存在差距。

### 技术约束

- 保持 Vue 3 + Element Plus 技术栈不变
- 复用现有 CSS Variables 系统
- 不引入新的 CSS 框架（如 Tailwind）
- 保持现有组件结构

### 利益相关者

- 产品用户：期望更流畅、更现代的使用体验
- 开发团队：需要可维护的设计系统

## Goals / Non-Goals

**Goals:**

1. 建立统一的设计语言和视觉规范
2. 实现流畅的微交互和动画系统
3. 优化核心页面的视觉层次和细节
4. 完善暗色模式的视觉体验
5. 提升整体产品质感和品牌感

**Non-Goals:**

- 不重构组件架构或状态管理
- 不引入新的 UI 框架或设计系统
- 不修改后端 API
- 不开发新功能
- 不做移动端适配

## Decisions

### 1. 设计语言：Calm Design + Bento Grid

**决定**：采用 Calm Design（冷静设计）理念，结合 Bento Grid 布局模式。

**理由**：
- Calm Design 强调默认隐藏非必要元素，让用户专注于当前任务
- Bento Grid 提供灵活的不对称卡片布局，自然形成视觉层级
- 这是 2026 年 SaaS 产品的主流趋势，Linear、Notion 均采用此理念

**替代方案**：
- 传统固定布局：过于死板，缺乏现代感
- 完全自由布局：实现复杂度高，维护成本大

### 2. 动画系统：意图驱动

**决定**：建立三层动画系统（反馈/过渡/惊喜），每个动画都有明确目的。

**动画规范**：
```css
/* 时长 */
--duration-fast: 150ms;   /* 按钮悬停、状态切换 */
--duration-normal: 200ms; /* 列表项、模态框 */
--duration-slow: 300ms;   /* 页面切换、展开动画 */

/* 缓动函数 */
--ease-out: cubic-bezier(0.0, 0.0, 0.2, 1);
--ease-in-out: cubic-bezier(0.4, 0.0, 0.2, 1);
--spring: cubic-bezier(0.175, 0.885, 0.32, 1.275);
```

**理由**：
- 动画应服务于用户理解，而非纯装饰
- 统一的时长和缓动函数确保一致性

### 3. 间距系统：8点网格

**决定**：严格遵循 8 点网格系统。

**间距规范**：
```
基础单位: 4px
常用间距: 4, 8, 12, 16, 24, 32, 48, 64px
组件内边距: 12-16px
卡片间距: 16-24px
区块间距: 32-48px
```

**理由**：
- 8 点网格是行业标准，确保视觉节奏一致
- 便于计算和记忆，减少随意取值

### 4. 登录页：品牌强化 + 零摩擦

**决定**：重新设计登录页，强化品牌感，简化表单流程。

**设计要点**：
- 左侧品牌展示区（渐变背景 + 产品特性）
- 右侧简洁表单（单列布局）
- SSO 按钮优先（Google、GitHub）
- 入场动画（品牌 logo 淡入 + 表单滑入）

**理由**：
- 登录页是产品的第一印象
- 零摩擦设计降低用户流失

### 5. 导航系统：可折叠 + 命令面板

**决定**：
- 侧边栏支持折叠/展开，默认展开
- 添加全局命令面板（Cmd/Ctrl + K）

**侧边栏设计**：
```
宽度: 240px（展开）/ 64px（折叠）
悬停: 背景色变浅 + 左侧 3px 品牌色指示条
选中: 背景色加深 + 品牌色文字
过渡: 150ms ease-out
```

**命令面板**：
- 居中模态框，宽度 560px
- 模糊搜索 + 键盘导航
- 分组显示（Recent、Actions、Navigation）

**理由**：
- 可折叠侧边栏减少干扰，符合 Calm Design
- 命令面板是现代 SaaS 标配，提升效率用户满意度

### 6. 任务列表：多视图 + 行内编辑

**决定**：
- 保持卡片/看板/表格三种视图
- 添加行内编辑能力
- 优化拖放交互反馈

**列表视图**：
- 行高: 48px
- 悬停显示操作按钮
- 优先级使用左侧色条

**看板视图**：
- 列宽: 300px
- 卡片间距: 12px
- 拖动时卡片旋转 1° + 阴影加深

**理由**：
- 多视图满足不同场景需求
- 行内编辑减少上下文切换
- 精细的拖放反馈提升操作信心

### 7. 模态框：侧边抽屉优先

**决定**：任务详情使用侧边抽屉，新建任务使用居中模态框。

**侧边抽屉**：
- 宽度: 480px
- 从右滑入，300ms ease-out
- 保持主界面上下文可见

**新建任务模态框**：
- 宽度: 520px
- 居中显示
- 表单 + 底部操作栏

**理由**：
- 侧边抽屉保持工作连续性
- 新建任务使用模态框聚焦用户注意力

### 8. 色彩系统：扩展中性色

**决定**：扩展现有色彩系统，增加中性色层级。

**新增中性色**：
```css
/* 浅色模式 */
--gray-50: #F9FAFB;  /* 背景 */
--gray-100: #F3F4F6; /* 次级背景 */
--gray-200: #E5E7EB; /* 边框 */
--gray-300: #D1D5DB; /* 禁用 */
--gray-400: #9CA3AF; /* 辅助文本 */
--gray-500: #6B7280; /* 次级文本 */
--gray-600: #4B5563; /* 正文 */
--gray-700: #374151; /* 强调 */
--gray-800: #1F2937; /* 标题 */
--gray-900: #111827; /* 主要文本 */

/* 暗色模式 */
--dark-bg-primary: #0F172A;
--dark-bg-secondary: #1E293B;
--dark-bg-tertiary: #334155;
```

**理由**：
- 丰富的中性色层级支持精细的视觉层次
- 暗色模式需要独立的背景层次

## Risks / Trade-offs

### 风险 1：动画性能影响低端设备

**风险**：复杂动画可能在低端设备上造成卡顿。

**缓解措施**：
- 使用 CSS transform 和 opacity（GPU 加速）
- 提供 `prefers-reduced-motion` 媒体查询支持
- 避免同时触发多个复杂动画

### 风险 2：Element Plus 样式覆盖复杂

**风险**：Element Plus 组件样式覆盖可能产生冲突。

**缓解措施**：
- 使用 CSS Variables 统一管理
- 优先使用 Element Plus 提供的主题定制接口
- 必要时使用 `::v-deep` 精准覆盖

### 风险 3：暗色模式适配遗漏

**风险**：部分组件可能遗漏暗色模式适配。

**缓解措施**：
- 建立暗色模式测试清单
- 使用 CSS Variables 确保所有颜色可切换
- 开发阶段强制暗色模式测试

### Trade-off：不引入 Tailwind CSS

**权衡**：保持现有 CSS Variables 方案，不迁移到 Tailwind。

**理由**：
- 迁移成本高，影响开发进度
- 现有方案已能满足需求
- 团队对 CSS Variables 更熟悉
