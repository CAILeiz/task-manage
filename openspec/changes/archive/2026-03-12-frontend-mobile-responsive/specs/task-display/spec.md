## MODIFIED Requirements

### REQ-001: 任务描述展示

**优先级**: P0

任务列表/卡片中必须展示任务的描述内容。**移动端适配描述截断和触摸查看。**

**验收标准**:
- [ ] 卡片视图显示完整描述文本
- [ ] 描述为空时显示"暂无描述"占位
- [ ] 描述过长时自动截断，hover显示完整内容
- [ ] 描述支持多行文本展示
- [ ] **移动端：描述超过 2 行时截断，点击展开查看完整内容**

#### Scenario: 移动端描述展开
- **WHEN** 用户在移动端点击被截断的描述
- **THEN** 展开显示完整描述内容

### REQ-002: 卡片式任务展示

**优先级**: P1

提供卡片视图替代纯表格展示，提升视觉层次感。**移动端单列布局适配。**

**验收标准**:
- [ ] 任务以独立卡片形式展示
- [ ] 卡片显示标题、描述、优先级、截止日期、状态
- [ ] 卡片左边框颜色标识优先级（红=高，黄=中，绿=低）
- [ ] hover时卡片有轻微上浮动画
- [ ] 支持列表/卡片视图切换
- [ ] **移动端（< 768px）：单列布局，卡片宽度 100%**
- [ ] **移动端：卡片内边距减少，触摸目标增大**

**组件规格**:
- 组件名: `TaskCard.vue`
- Props: `task: Task`
- Events: `@edit`, `@delete`, `@status-change`
- **新增 Props**: `compact: boolean` （移动端紧凑模式）

#### Scenario: 移动端卡片布局
- **WHEN** 屏幕宽度小于 768px
- **THEN** 任务卡片以单列排列，间距 12px

#### Scenario: 移动端卡片触摸
- **WHEN** 用户触摸任务卡片
- **THEN** 卡片显示触摸高亮效果

### REQ-003: 看板视图

**优先级**: P2

提供看板视图，按状态分组展示任务，支持拖拽。**移动端隐藏看板视图。**

**验收标准**:
- [ ] 三列布局：待开始、进行中、已完成
- [ ] 任务卡片在列内展示
- [ ] 支持拖拽任务卡片到不同列
- [ ] 拖拽后自动更新任务状态
- [ ] 列顶部显示任务计数
- [ ] **移动端（< 768px）：隐藏看板视图选项**

#### Scenario: 移动端视图切换
- **WHEN** 屏幕宽度小于 768px
- **THEN** 视图切换仅显示卡片视图和列表视图选项

## Technical Constraints

### TC-001: 性能要求
- 任务列表渲染时间 < 100ms（100条以内）
- 卡片动画帧率 ≥ 60fps
- **移动端滚动帧率 ≥ 60fps**

### TC-002: 兼容性
- 支持 Chrome、Firefox、Safari 最新版本
- 响应式设计支持 320px 以上屏幕
- **移动端支持 iOS Safari 和 Android Chrome**

## ADDED Requirements

### Requirement: 移动端筛选抽屉

移动端筛选器 SHALL 以底部抽屉形式展示。

#### Scenario: 打开筛选抽屉
- **WHEN** 移动端用户点击筛选按钮
- **THEN** 筛选器从底部滑入，带有遮罩层

#### Scenario: 筛选器内容
- **WHEN** 筛选抽屉打开
- **THEN** 显示优先级、状态、日期筛选选项及操作按钮

### Requirement: 移动端分页简化

移动端分页组件 SHALL 简化显示。

#### Scenario: 简化分页
- **WHEN** 屏幕宽度小于 768px
- **THEN** 分页仅显示上一页/下一页按钮和当前页码
