# Sidebar Navigation Specification

## Capability

侧边栏导航，提供快捷视图入口、标签筛选和**任务统计数量的实时更新**。

## ADDED Requirements

### REQ-005: 任务统计数量实时更新

**优先级**: P0

任务状态变更后，侧边栏的各分类任务数量 MUST 自动更新。

**验收标准**:
- [ ] 数量包含：全部、今日、过期、即将到期、无期限、已完成
- [ ] 任务完成/取消完成后，数量立即更新
- [ ] 新增/删除任务后，数量立即更新
- [ ] 通过 WebSocket 推送更新，无需刷新页面
- [ ] 数量更新有过渡动画

#### Scenario: 任务完成后数量更新
- **WHEN** 用户标记任务为完成
- **THEN** "已完成"数量+1
- **AND** 如果任务原本在"今日"，则"今日"数量-1
- **AND** "全部"数量保持不变（已完成仍计入总数）

#### Scenario: 取消完成后数量更新
- **WHEN** 用户取消任务完成状态
- **THEN** "已完成"数量-1
- **AND** 根据任务截止日期重新计算其他分类数量

#### Scenario: WebSocket 推送更新
- **WHEN** 任务状态发生变更
- **THEN** 服务端推送 counts_update 消息
- **AND** 前端接收后更新侧边栏显示

### REQ-006: 数量徽章动态效果

**优先级**: P2

数量变化时 MUST 有视觉动效提示用户。

**验收标准**:
- [ ] 数量增加时徽章有放大动画
- [ ] 数量减少时徽章有缩小动画
- [ ] 动画持续时间 200ms
- [ ] 动画不阻塞用户操作

#### Scenario: 数量增加动画
- **WHEN** 分类任务数量从 2 变为 3
- **THEN** 徽章显示放大后恢复的动画效果
