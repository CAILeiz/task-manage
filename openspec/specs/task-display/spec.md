# Task Display Specification

## Capability

优化任务展示方式，支持描述内容展示、卡片视图和看板视图。

## Requirements

### REQ-001: 任务描述展示

**优先级**: P0

任务列表/卡片中必须展示任务的描述内容。

**验收标准**:
- [ ] 卡片视图显示完整描述文本
- [ ] 描述为空时显示"暂无描述"占位
- [ ] 描述过长时自动截断，hover显示完整内容
- [ ] 描述支持多行文本展示

**UI规格**:
```
┌────────────────────────────────────────────────┐
│ 完成项目报告                          🔴 高优先级│
│ ────────────────────────────────────────────   │
│ 需要包含Q4数据分析、用户增长趋势和下季度规划    │  ← 描述
│                                                │
│ 📅 截止: 今天 18:00    🏷 工作    📊 进行中     │
└────────────────────────────────────────────────┘
```

### REQ-002: 卡片式任务展示

**优先级**: P1

提供卡片视图替代纯表格展示，提升视觉层次感。

**验收标准**:
- [ ] 任务以独立卡片形式展示
- [ ] 卡片显示标题、描述、优先级、截止日期、状态
- [ ] 卡片左边框颜色标识优先级（红=高，黄=中，绿=低）
- [ ] hover时卡片有轻微上浮动画
- [ ] 支持列表/卡片视图切换

**组件规格**:
- 组件名: `TaskCard.vue`
- Props: `task: Task`
- Events: `@edit`, `@delete`, `@status-change`

### REQ-003: 看板视图

**优先级**: P2

提供看板视图，按状态分组展示任务，支持拖拽。

**验收标准**:
- [ ] 三列布局：待开始、进行中、已完成
- [ ] 任务卡片在列内展示
- [ ] 支持拖拽任务卡片到不同列
- [ ] 拖拽后自动更新任务状态
- [ ] 列顶部显示任务计数

## Technical Constraints

### TC-001: 性能要求
- 任务列表渲染时间 < 100ms（100条以内）
- 卡片动画帧率 ≥ 60fps

### TC-002: 兼容性
- 支持 Chrome、Firefox、Safari 最新版本
- 响应式设计支持 1280px 以上屏幕

## Out of Scope

- 富文本描述编辑器
- 任务评论功能
- 任务附件功能
