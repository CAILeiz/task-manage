# User Profile Menu Specification

## Capability

用户头像hover下拉菜单，展示用户详细信息及操作入口。

## Requirements

### REQ-001: 用户头像展示

**优先级**: P0

在页面顶部显示用户头像和用户名。

**验收标准**:
- [ ] 显示圆形头像（32px）
- [ ] 头像旁显示用户名
- [ ] 无头像时显示用户名首字母大写
- [ ] 支持自定义头像URL（后端扩展）

### REQ-002: 下拉菜单触发

**优先级**: P0

hover或点击用户区域触发下拉菜单。

**验收标准**:
- [ ] hover触发下拉菜单显示
- [ ] 菜单在用户区域下方右对齐显示
- [ ] 鼠标移出菜单区域后自动关闭
- [ ] 支持点击触发（移动端适配）

### REQ-003: 用户信息展示

**优先级**: P0

下拉菜单中展示用户详细信息。

**验收标准**:
- [ ] 显示用户名
- [ ] 显示邮箱地址
- [ ] 显示注册时间（格式：YYYY-MM-DD）
- [ ] 信息区域禁用点击（仅展示）

**UI规格**:
```
┌─────────────────────────────────┐
│ 用户名: zhang_san               │
│ 邮箱: zhang@example.com         │
│ 注册时间: 2024-01-15            │
├─────────────────────────────────┤
│ ⚙️ 个人设置                      │
│ 🚪 退出登录                     │
└─────────────────────────────────┘
```

### REQ-004: 操作入口

**优先级**: P0

下拉菜单提供快捷操作入口。

**验收标准**:
- [ ] 提供"个人设置"入口（预留，暂跳转到404或提示）
- [ ] 提供"退出登录"入口
- [ ] 退出登录需要二次确认
- [ ] 退出后跳转到登录页

## Technical Constraints

### TC-001: 组件要求
- 使用 Element Plus `el-dropdown` 组件
- 使用 Element Plus `el-avatar` 组件
- 使用 Element Plus `el-descriptions` 组件

### TC-002: 数据来源
- 用户信息从 `authStore.user` 获取
- 需要 `username`, `email`, `createdAt` 字段

## Out of Scope

- 头像上传功能
- 用户信息编辑功能
- 密码修改功能
